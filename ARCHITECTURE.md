## 전체 그림

```
┌─────────────────────────────────────────────────────────────────┐
│  외부 소스 (Pull)                                                  │
│  Hacker News API · Reddit .json · RSS/Atom(Rust·Go·arXiv …)       │
└───────────────┬─────────────────────────────────────────────────┘
                │ collectAll()  ── 소스별 격리(하나 죽어도 전체 안 죽음)
                ▼
┌─────────────────────────────────────────────────────────────────┐
│  server/ (Hono + SQLite)                                          │
│                                                                   │
│   [스케줄러] node-cron ── 도착 N분 전 트리거                        │
│        │                                                          │
│        ▼  generateBriefing()                                      │
│   수집 → dedup(seen 제외·중복병합·과거컷) → 화제도 z-정규화          │
│        → Haiku 8장르 분류 → 랭킹(필독3 + 더보기N)                   │
│        → 요약(본문 있으면 Haiku·없으면 Sonnet) → SQLite 저장        │
│        │                                                          │
│        ▼  도착 정각                                                │
│   [스케줄러] sendPushToAll() ──────────────────┐                  │
│                                                │                  │
│   [REST API] /api/briefing /read /dashboard …  │                  │
│        ▲                                       │                  │
└────────┼───────────────────────────────────────┼─────────────────┘
         │ fetch (JWT)                            │ Web Push (VAPID)
         │                                        ▼
┌────────┴───────────────────────────────────────────────────────┐
│  web/ (SvelteKit PWA)                                            │
│  로그인 → 브리핑 뷰 → 대시보드 → 설정                              │
│  Service Worker(push-sw.js) ── 백그라운드에서 푸시 수신·표시        │
└─────────────────────────────────────────────────────────────────┘
```

### 수집창·갱신 풀

- **48h 단일 수집창** — `dedup(maxAgeDays=2)`. 별도 24h 컷은 두지 않는다(평가 지연으로 구제한
  글이 경계에 다시 걸리면 무의미). 신선도는 시간 컷이 아니라 랭킹(화제도순)이 보장.
- **`candidate_pool`** — 수집 때 첫 묶음(필독3+더보기7)만 요약해 `items`에 저장하고, 나머지 후보
  전체는 휘발성 풀에 보관(요약 안 함). `POST /briefing/:id/more`(=`loadMore`)가 다음 N건을 **그때**
  요약(본문 없는 외부링크는 `fetchArticleText` 크롤링)해 `items`로 승격하고 `shown=1` 마킹.
  다음 4:30 수집이 풀을 통째 교체. → `items`·`feedback`·대시보드가 '안 본 후보'로 오염되지 않음.

### 배포

Railway 단일 서비스(`railway.toml`+`.nvmrc`). server가 `web/build`까지 same-origin 서빙,
SQLite는 영속 볼륨(`DB_PATH=/data/...`)에 둬 재배포에도 유지. cron은 컨테이너 내 node-cron.
